# 变更：/team 人员管理表格（CRUD）

## 为什么

当前 `/team` 页面为占位内容，无法进行团队人员信息的管理。为了支持基础的团队管理工作流，需要提供一个可增删改查的人员管理表格，覆盖常用字段，并遵循本仓库既有的工程化与代码规范。

## 变更内容

- 在 `/team` 页面提供“人员管理”表格视图
- 支持人员信息的 **创建 / 查看（列表） / 更新 / 删除（CRUD）**
- 人员字段：
  - 姓名
  - 性别
  - 身份证号
  - 职位
  - 所属部门
  - 职级
- 数据来源为 **前端 mock**（无需对接后端 API）
- 身份证号要求 **唯一**
- 性别/所属部门/职级为 **固定枚举**（表单使用选择控件）

## 不在本次范围（明确非目标）

- 与真实后端/数据库对接的人员管理 API
- 复杂权限体系（如多角色授权、字段级权限）
- 复杂表格能力（高级筛选、批量操作、导入导出、服务端分页）
- 本地持久化与跨刷新数据恢复（如需再单独提案）

## 影响

- **新增规范 capability**：`team`
- **受影响代码（实施阶段）**（预估）：
  - `src/views/TeamView.vue`（从占位页升级为人员管理页面）
  - `src/stores/*`（新增 team/member 相关 store）
  - `src/components/*`（复用或新增表格、表单、确认弹窗等组件）
  - `src/composables/*`、`src/utils/*`（抽取可测试逻辑，如脱敏/校验）
  - `src/__test__/*`（新增/扩展单测）

## 已确认（范围约束）

- **数据来源**：mock 即可，无需后端 API
- **身份证展示**：无需脱敏；但身份证号必须唯一
- **枚举字段**：部门/职级/性别为固定枚举
